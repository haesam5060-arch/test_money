# 글로벌 개발 원칙

## 1. 데이터 기반 개발
- 모든 개발은 수익화를 목표로 하며, 데이터를 기반으로 의사결정한다.
- 추측이나 감이 아닌 실제 데이터와 통계로 검증된 로직만 적용한다.
- 새로운 기능이나 전략 변경 시 반드시 백테스트 결과를 근거로 제시한다.

## 2. 개발 완료 후 반드시 검증
- 기능 개발이 완료되면 반드시 검증 과정을 거친다.
- 검증 없이 완료로 간주하지 않는다.
- 단위 테스트, 통합 테스트, 실제 데이터 기반 검증을 수행한다.
- 수치가 관련된 로직은 경계값과 예외 케이스까지 검증한다.

## 3. 기획 우선 개발
- 모든 개발 요청은 기획 단계를 먼저 거쳐 사용자에게 확인을 받은 후 구현한다.
- 구현 전에 목적, 접근 방식, 영향 범위를 정리하여 사용자 승인을 받는다.
- 승인 없이 코드 작성을 시작하지 않는다.

## 4. 장애 대응 시 전체 점검
- 이상동작, 미동작, 장애 등의 요청이 들어오면 해당 건만 수정하지 않는다.
- 반드시 유사한 기능 및 전체 시스템으로 확장하여 동일한 문제가 없는지 전수 점검한다.
- 근본 원인(root cause)을 파악하고, 같은 패턴의 잠재적 문제를 함께 해결한다.

## 5. 코드 품질
- 금융 데이터를 다루므로 계산 정확성이 최우선이다. 부동소수점 오차에 주의한다.
- 외부 API(KIS, 네이버) 호출 시 에러 핸들링과 재시도 로직을 반드시 포함한다.
- 민감 정보(API 키, 계좌번호)는 절대 로그나 응답에 노출하지 않는다.
- 기존 코드 스타일과 패턴을 따른다. 불필요한 리팩토링을 하지 않는다.

## 6. 안전 우선
- 실제 자금이 걸린 시스템이므로, 파괴적 변경은 반드시 확인 후 실행한다.
- 모의투자로 먼저 검증하고, 충분한 검증 후에만 실전에 적용한다.
- 주문/매매 관련 로직 변경 시 기존 포지션에 미치는 영향을 반드시 분석한다.
